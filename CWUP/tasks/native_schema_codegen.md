# Native Schema Code Generation (C++ Headers)

## Goal
Extend `tools/generate_sml_element_docs.gd` to emit C++ header files parallel to
the existing `ForgeRunner/Generated/Schema*.cs` files, so that
`ForgeRunner.Native` has the same schema knowledge (types, properties, events,
context-properties, aliases) without hardcoding it in `forge_ui_builder.cpp`.

## Context
`tools/generate_sml_element_docs.gd` is the single source of truth generator.
It reads all `tools/specs/*.gd` specs and currently emits:

| Generated file (C#) | Content |
|---|---|
| `ForgeRunner/Generated/SchemaTypes.cs` | type hierarchy (Name, Parent, IsCollection) |
| `ForgeRunner/Generated/SchemaProperties.cs` | per-type property map (SmlName → GodotName + ValueType) |
| `ForgeRunner/Generated/SchemaEvents.cs` | per-type event map (SmsName → GodotSignal + params) |
| `ForgeRunner/Generated/SchemaContextProperties.cs` | attached-property defs |
| `ForgeRunner/Generated/SchemaLayoutAliases.cs` | size/position aliases |
| `ForgeRunner/Generated/SchemaLayoutDefaults.cs` | MenuBar layout defaults |

The C++ builder (`forge_ui_builder.cpp`) currently hardcodes much of this or
omits it entirely. Any spec change requires a manual C++ update.

## Output Files (C++ Headers)

Add a new `NATIVE_GENERATED_DIR` variable pointing to
`ForgeRunner.Native/src/generated/` and emit matching headers:

```
ForgeRunner.Native/src/generated/schema_types.h
ForgeRunner.Native/src/generated/schema_properties.h
ForgeRunner.Native/src/generated/schema_events.h
ForgeRunner.Native/src/generated/schema_context_properties.h
ForgeRunner.Native/src/generated/schema_layout_aliases.h
ForgeRunner.Native/src/generated/schema_layout_defaults.h
```

All files carry `// <auto-generated — do not edit>` and `#pragma once`.

## C++ Data Structures

### schema_types.h
```cpp
#pragma once
#include <string_view>
struct SchemTypeDef { std::string_view name; std::string_view parent; bool is_collection; };
inline constexpr SchemaTypeDef kSchemaTypes[] = {
    {"AcceptDialog",          "Window",    false},
    {"Button",                "BaseButton",false},
    // ... generated from SchemaTypes
};
```

### schema_properties.h
```cpp
#pragma once
#include <string_view>
struct SchemaPropDef { std::string_view type_name; std::string_view sml_name;
                       std::string_view godot_name; std::string_view value_type; };
inline constexpr SchemaPropDef kSchemaProperties[] = {
    {"Button", "text", "text", "string"},
    {"Button", "disabled", "disabled", "bool"},
    // ... generated
};
```

### schema_events.h
```cpp
#pragma once
#include <string_view>
struct SchemaEventDef { std::string_view type_name; std::string_view sms_name;
                        std::string_view godot_signal; };
// param arrays omitted for simplicity; extend if needed
inline constexpr SchemaEventDef kSchemaEvents[] = {
    {"BaseButton", "pressed",    "pressed"},
    {"BaseButton", "toggled",    "toggled"},
    // ...
};
```

### schema_context_properties.h
```cpp
inline constexpr SchemaContextPropDef kSchemaContextProperties[] = {
    {"TabContainer",      "*", "title", "string", "tabTitle"},
    {"DockingContainer",  "*", "title", "string", "tabTitle"},
};
```

### schema_layout_aliases.h / schema_layout_defaults.h
Analogous to C# versions; `constexpr` structs.

## Generator Changes (generate_sml_element_docs.gd)

Add at the end of `_run()`:
```gdscript
NATIVE_GENERATED_DIR = REPO_ROOT + "/ForgeRunner.Native/src/generated"
DirAccess.make_dir_recursive_absolute(NATIVE_GENERATED_DIR)
_emit_cpp_schema_types(all_type_defs)
_emit_cpp_schema_properties(all_prop_defs)
_emit_cpp_schema_events(all_event_defs)
_emit_cpp_schema_context_properties(all_context_prop_defs)
_emit_cpp_schema_layout_aliases()
_emit_cpp_schema_layout_defaults()
```

Each `_emit_cpp_*` function mirrors the existing `_emit_cs_*` functions in style.

## Usage in forge_ui_builder.cpp

After generation, `UiBuilder::create_control()` can use `kSchemaTypes` for
unknown-node fallback, `UiBuilder::apply_props()` can iterate `kSchemaProperties`
for the generic property setter, and `UiBuilder::bind_signals()` can iterate
`kSchemaEvents` to auto-connect SMS event handlers — eliminating manual maintenance.

## Build Integration

Add `generated/` to `.gitignore` for `ForgeRunner.Native/src/generated/` so the
files are never committed (generated on `./run.sh docs`).

## Acceptance Criteria
- `./run.sh docs` produces all 6 C++ header files under
  `ForgeRunner.Native/src/generated/`.
- `forge_ui_builder.cpp` includes `generated/schema_properties.h` and uses it
  in `apply_props()` for at least the generic Godot-property path.
- Adding a new spec property in `tools/specs/*.gd` and running `./run.sh docs`
  automatically updates both `SchemaProperties.cs` and `schema_properties.h`.
- Generated files are listed in `.gitignore` and never checked in.
